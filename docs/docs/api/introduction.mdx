# Introduction

Before using the Züs JS SDK, you need to initialize the SDK first. See [Quick Start guide](/docs/getting-started/quick-start) for more details.

After initializing the SDK, you can use the Züs JS SDK to perform various operations such as creating and managing allocations, uploading and downloading files, executing smart contracts, and more.

## SDK Methods Index

In this guide, we will introduce you to all the available methods provided by the Züs JS SDK.

### Wallet Methods

The wallet methods allow you to create and manage wallets, set the wallet, and get wallet information.

| Method | Description |
| --- | --- |
| [`createWallet`](/docs/api/wallet#createwallet) | Create a new wallet. Also, helps to recover a wallet using its mnemonic phrase. |
| [`createWalletKeys`](/docs/api/wallet#createwalletkeys) | Create a new wallet keys and mnemonic. |
| [`getPublicEncryptionKey`](/docs/api/wallet#getpublicencryptionkey) | Get the public encryption key by wallet mnemonic. | 
| [`isWalletId`](/docs/api/wallet#iswalletid) | Check whether thef wallet id is valid. |
| [`send`](/docs/api/wallet#send) | Send token from one wallet to another. |
| [`getWalletBalance`](/docs/api/wallet#getwalletbalance) | Get the balance of a wallet. |
| [`getUSDRate`](/docs/api/wallet#getusdrate) | Get the USD rate by token symbol. (e.g. `zcn`, `eth`) |

### Burn & Mint Methods

The `initBridge` method must be called to initialize the bridge client before using the any of the burn and mint methods.

| Method | Description |
| --- | --- |
| [`initBridge`](/docs/api/burn-mint#initbridge) | Initialize the bridge for calling burn and mint functions. |
| [`burnZCN`](/docs/api/burn-mint#burnzcn) | Bridge method to burn ZCN tokens. |
| [`mintZCN`](/docs/api/burn-mint#mintzcn) | Bridge method to mint ZCN tokens. |
| [`getMintWZCNPayload`](/docs/api/burn-mint#getmintwzcnpayload) | Bridge method to get *Mint WZCN Payload* for a given burn transaction hash. |
| [`getUnprocessedWZCNBurnEvents`](/docs/api/burn-mint#getunprocessedwzcnburnevents) | Get all unprocessed WZCN burn events from the Ethereum network. |
| [`getProcessedZCNBurnTickets`](/docs/api/burn-mint#getprocessedzcnburntickets) | Get all processed ZCN burn tickets burned for a certain Ethereum address. |
| [`estimateMintWZCNGasAmount`](/docs/api/burn-mint#estimatemintwzcngasamount) | Perform gas amount estimation for the given Mint WZCN transaction. |
| [`estimateBurnWZCNGasAmount`](/docs/api/burn-mint#estimateburnwzcngasamount) | Perform gas amount estimation for the given burn WZCN transaction. |
| [`estimateGasPrice`](/docs/api/burn-mint#estimategasprice) | Perform gas estimation for the given transaction using Alchemy enhanced API returning approximate final gas fee. |

### Allocation Storage Methods

The allocation storage methods allow you to create, update, and manage allocations.

| Method | Description |
| --- | --- |
| [`createAllocation`](/docs/api/allocation#createallocation) | Creates an allocation with the specified allocation terms and preferred blobber IDs. |
| [`getAllocationMinLock`](/docs/api/allocation#getallocationminlock) | Retrieves the minimum lock value for the allocation creation. |
| [`listAllocations`](/docs/api/allocation#listallocations) | List all the allocations. |
| [`getAllocation`](/docs/api/allocation#getallocation) | Gets allocation details for a given `allocationId` from cache or the blockchain. |
| [`reloadAllocation`](/docs/api/allocation#reloadallocation) | Re-fetches allocation details using `allocationId` from blockchain and updates cache.  |
| [`getAllocationWith`](/docs/api/allocation#getallocationwith) | Get allocation details of a free allocation or a shared allocation details by using an authTicket. |
| [`freezeAllocation`](/docs/api/allocation#freezeallocation) | Freeze an allocation so that the allocation data can no longer be modified. |
| [`cancelAllocation`](/docs/api/allocation#cancelallocation) | Cancel an Allocation |
| [`updateAllocation`](/docs/api/allocation#updateallocation) | Update the allocation settings |
| [`updateAllocationWithRepair`](/docs/api/allocation#updateallocationwithrepair) | Updates your allocation settings and repairs the allocation if any blobber was replaced or added |
| [`getUpdateAllocationMinLock`](/docs/api/allocation#getupdateallocationminlock) | Retrieves the minimum lock value for the allocation after update. |
| [`updateForbidAllocation`](/docs/api/allocation#updateforbidallocation) | UpdateForbidAllocation updates the permissions of an allocation, given the permission parameters in a forbid-first manner. |
| [`getUpdateAllocTicket`](/docs/api/allocation#getupdateallocticket) | This method is used to sign an "Update Allocation ticket". This ticket is needed to allow someone else to run update transaction for your allocation on your terms. |
| [`transferAllocation`](/docs/api/allocation#transferallocation) | Transfer the ownership of an allocation to a new owner |
| [`checkAllocStatus`](/docs/api/allocation#checkallocstatus) | Check the health status of the allocation.|
| [`skipStatusCheck`](/docs/api/allocation#skipstatuscheck) | Skips the `checkAlloctionStatus` health status check of the allocation. |
| [`repairAllocation`](/docs/api/allocation#repairallocation) | Start a repair process for an allocation. |
| [`repairSize`](/docs/api/allocation#repairsize) | Retrieves the repair size for a specific path in an allocation. |
| [`createWorkers`](/docs/api/allocation#createworkers) | Manually create local workers that sync with the allocation. |
| [`terminateWorkers`](/docs/api/allocation#terminateworkers) | Remove local workers that sync with the allocation. This is useful when switching between allocations. |

### Blobber Methods

The blobber methods allow you to retrieve information, manage and interact with blobbers.

| Method | Description |
| --- | --- |
| [`getBlobbers`](/docs/api/blobber#getblobbers) | Retrieves a list of active blobbers details from the network |
| [`getBlobberIds`](/docs/api/blobber#getblobberids) | Get Blobber IDs from a list of blobber URLs |
| [`getAllocationBlobbers`](/docs/api/blobber#getallocationblobbers) | Retrieves a list of blobber IDs of blobber which match your allocation terms |
| [`updateBlobberSettings`](/docs/api/blobber#updateblobbersettings) | Updates the blobber settings |
| [`getContainers`](/docs/api/blobber#getcontainers) | GetContainers returns all the running containers in a given domain. |
| [`searchContainer`](/docs/api/blobber#searchcontainer) | searchContainer searches for a container with a given name in a given domain. |
| [`updateContainer`](/docs/api/blobber#updatecontainer) | UpdateContainer updates the given container ID with a new image ID in a given domain. |

### File Operation Methods

The file operation methods allow you to perform various operations on files.

| Method | Description |
| --- | --- |
| [`getLookupHash`](/docs/api/file#getlookuphash) | Generate lookup hash for a given file path and allocation ID |
| [`createDir`](/docs/api/file#createdir) | Create a folder on the blobbers |
| [`setUploadMode`](/docs/api/file#setuploadmode) | Sets the upload mode for modifying the upload speed and CPU usage |
| [`multiUpload`](/docs/api/file#multiupload) | Upload multiple files in a batch. Also, used to resume a paused upload. |
| [`pauseUpload`](/docs/api/file#pauseupload) | Pause the upload operation of the file. |
| [`cancelUpload`](/docs/api/file#cancelupload) | Cancel the upload operation of the file. |
| [`getFileStats`](/docs/api/file#getfilestats) | Gets the file stats |
| [`listObjects`](/docs/api/file#listobjects) | List the files for a given allocation ID and remote path |
| [`getFileMetaByName`](/docs/api/file#getfilemetabyname) | Get file metadata by name. (File Search) |
| [`multiDownload`](/docs/api/file#multidownload) | Download multiple files together |
| [`downloadBlocks`](/docs/api/file#downloadblocks) | Downloads a specified range of blocks from a file. |
| [`cancelDownloadBlocks`](/docs/api/file#canceldownloadblocks) | Cancels the download of a specified range of blocks from a file. |
| [`downloadDirectory`](/docs/api/file#downloaddirectory) | Download files in a directory. |
| [`cancelDownloadDirectory`](/docs/api/file#canceldownloaddirectory) | Cancel the download of a directory. |
| [`multiOperation`](/docs/api/file#multioperation) | Perform multiple file operations (copy, move, delete, create directory) together |
| [`deleteFile`](/docs/api/file#deletefile) | Deletes a file from an allocation. Only the owner of the allocation can delete a file. Prefer `multiOperation` for deleting multiple files. |
| [`share`](/docs/api/file#share) | Generate an authToken that provides the authorization to the holder to the specified file on the remotepath. |
| [`listObjectsFromAuthTicket`](/docs/api/file#listobjectsfromauthticket) | List objects from an auth ticket. It's useful for accessing a shared resource by a non-owner |

### Smart Contract Methods

The smart contract methods allow you to interact with smart contracts.

| Method | Description |
| --- | --- |
| [`makeSCRestAPICall`](/docs/api/smart-contract#makescrestapicall) | Issues a request to the public API of the smart contracts |
| [`getStakePoolInfo`](/docs/api/smart-contract#getstakepoolinfo) | Get information about the stake pool for the allocation |
| [`lockWritePool`](/docs/api/smart-contract#lockwritepool) | Lock given number of tokens for given duration in write pool |
| [`lockStakePool`](/docs/api/smart-contract#lockstakepool) | Stake number of tokens for a given provider given its type and ID |
| [`unlockStakePool`](/docs/api/smart-contract#unlockstakepool) | Unlock the write pool |
| [`collectRewards`](/docs/api/smart-contract#collectrewards) | Collect all rewards for a given delegate and provider pair. |
